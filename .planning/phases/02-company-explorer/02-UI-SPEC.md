## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inline padding |
| sm | 8px | Compact element spacing (badge-to-badge gap, filter chip gap) |
| md | 16px | Default element spacing (list row internal padding, filter bar padding) |
| lg | 24px | Section padding (detail pane section spacing) |
| xl | 32px | Layout gaps (list-pane to detail-pane gutter) |
| 2xl | 48px | Major section breaks (between detail-pane sections: firmographics / signals / personas) |
| 3xl | 64px | Page-level spacing (top-level page padding on wide viewports) |

Exceptions:
- Collapsed sidebar icon rail width: **48px** (multiple of 4, sized for icon-only nav items)
- List row minimum height: **48px** (touch/click target minimum for scan-and-triage rows, per EXPL-05)

## Typography

Dense, data-table-appropriate sizing (recall.ai Explorer Dashboard reference, not a marketing/content page) — 4 sizes, 2 weights.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | Regular (400) | 1.5 |
| Label | 12px | Regular (400) | 1.4 |
| Heading | 18px | Semibold (600) | 1.2 |
| Display | 24px | Semibold (600) | 1.2 |

Usage:
- **Body (14/400/1.5)**: list row primary text (company name in row), detail-pane body copy, filter/search input text
- **Label (12/400/1.4)**: badge text, "last updated" timestamps, firmographic field labels, table column headers
- **Heading (18/600/1.2)**: detail-pane section headers ("Firmographics", "Buying Signals", "Tech Stack", "Linked Personas")
- **Display (24/600/1.2)**: Company name in the detail-pane header (top of the open record)

No third weight (no bold/700) and no fifth size — do not introduce ad-hoc sizes for one-off elements; reuse these four.

## Color

60/30/10 split, confirmed against existing Astro app precedent (slate neutrals + indigo-600 accent already established in `src/pages/index.astro`, `src/pages/l/[code].astro`).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | white / slate-50 (`#f8fafc`) | Page background, detail-pane surface |
| Secondary (30%) | slate-100 / slate-200 (`#f1f5f9` / `#e2e8f0`) | Sidebar background, list-row hover, card/table borders, filter-bar background |
| Accent (10%) | indigo-600 (`#4f46e5`) | Active/selected nav item, active/selected list row indicator, active filter chip, links, focus rings |
| Destructive | Not applicable this phase | No delete/edit/destructive actions exist in Phase 2 (browse-only per COMP-01–04, EXPL-01–07) |

Accent reserved for: active/selected sidebar nav item (Companies/Key Personas), the selected list row's left-border/highlight indicator, active filter chip background, inline text links (none currently planned beyond nav), and focus rings on interactive elements. **Never** apply accent to static badges, section headers, or body text — accent marks interactivity/selection state only.

**Signal badges (single confirmed decision):** all four buying-signal types (financial cost pressure, no mature GBS/SSC org, new CFO/GBS head, transformation program announcement) share **one neutral "attention" badge style** — amber-100 background (`#fef3c7`) / amber-800 text (`#92400e`) — regardless of signal type. The signal type name is carried in the badge's label text, not color-coded per type. Do not introduce a distinct hue per signal type.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable this phase — there is no page-level CTA button. The primary interaction is a list-row click, which opens the record in the master-detail pane (EXPL-04). Do not add a "View" button on rows; the row itself is the affordance. |
| Empty state heading (filtered — no results match) | "No companies match your filters" |
| Empty state body (filtered) | "Try removing a filter or clearing your search." |
| Empty state heading (true empty — no data loaded) | "No companies yet" |
| Empty state body (true empty) | "Company data will appear here once the seed dataset is loaded." |
| Error state | "Couldn't load companies" — "Something went wrong fetching this data. Try refreshing the page." |
| Destructive confirmation | Not applicable — no destructive actions exist in this phase |

Note: EXPL-06 (explicit empty/loading/error-state handling) is formally scoped to Phase 4, but Phase 2's list and detail pane must not visibly break in these states even before Phase 4's hardening pass — use the copy above as the baseline now rather than leaving placeholder/undefined states.
