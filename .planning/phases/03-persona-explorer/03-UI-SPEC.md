## Spacing Scale

Inherited verbatim from `02-UI-SPEC.md`, confirmed against shipped code (`company-detail.tsx`: `p-8`, `space-y-12`, `space-y-2`, `mb-4`, `gap-2`/`gap-3`/`gap-4`; `company-list.tsx`: `min-h-48`, `min-h-12`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inline padding |
| sm | 8px | Compact element spacing (badge-to-badge gap, filter chip gap) |
| md | 16px | Default element spacing (list row internal padding, filter bar padding) |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps (list-pane to detail-pane gutter) |
| 2xl | 48px | Major section breaks (between detail-pane sections) |
| 3xl | 64px | Page-level spacing |

Exceptions (inherited, apply identically to Persona detail's sections ŌĆö Role & Seniority / Current Company / Career History / Contact Info use the same `space-y-12` outer rhythm and `mb-4` section-header gap as Company detail's Firmographics/Tech Stack/Signals/Linked Personas):
- List row minimum height: **48px** (`min-h-12` = 48px, touch/click target minimum, same as Company list rows)
- Empty/error-state card minimum height: **192px** (`min-h-48`, same as Company list)

## Typography

Inherited verbatim ŌĆö exact bracket-value Tailwind classes confirmed in shipped `company-detail.tsx`/`company-list.tsx` (`text-[Npx] font-{weight} leading-[ratio]`):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | Regular (400) | 1.5 |
| Label | 12px | Regular (400) | 1.4 |
| Heading | 18px | Semibold (600) | 1.2 |
| Display | 24px | Semibold (600) | 1.2 |

Usage on Persona surfaces (direct analogy to Company's usage):
- **Body (14/400/1.5)**: list row primary text (persona name), detail-pane body copy (title, career-history entries, contact-info values), filter/search input text
- **Label (12/400/1.4)**: badge text, seniority/"last updated"-style timestamps, `FirmographicField`-style field labels (reused verbatim per `03-CONTEXT.md` D-04/`03-RESEARCH.md` Pattern 4), table column headers
- **Heading (18/600/1.2)**: detail-pane section headers ŌĆö "Role & Seniority", "Current Company", "Career History", "Contact Info" (mirrors "Firmographics"/"Buying Signals" exactly)
- **Display (24/600/1.2)**: Persona name in the detail-pane header (top of the open record), same role as Company name in `company-detail.tsx:50`

No third weight, no fifth size ŌĆö Persona surfaces introduce zero new typographic roles.

## Color

Inherited verbatim ŌĆö confirmed against shipped Tailwind utility classes (slate neutrals + indigo-600 accent + amber signal badges; note `components.json`'s `baseColor: "neutral"` preset is not reflected in actual class usage, which uses literal `slate-*` classes throughout ŌĆö this is Phase 2's shipped reality, not a Phase 3 decision, and Phase 3 follows the same literal-class convention for consistency):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | white / slate-50 (`#f8fafc`) | Page background, detail-pane surface |
| Secondary (30%) | slate-100 / slate-200 (`#f1f5f9` / `#e2e8f0`) | Sidebar background, list-row hover, card/table borders, filter-bar background |
| Accent (10%) | indigo-600 (`#4f46e5`) | Active/selected sidebar nav item (now includes "Key Personas" per D-11's resolution ŌĆö `app-sidebar.tsx` converts from hardcoded to `usePathname()`-driven per `03-RESEARCH.md` Pattern 5), selected Persona list row's left-border indicator (`border-l-2 border-l-indigo-600 bg-indigo-50/50`, same class shape as `company-list.tsx:136`), active filter chip background, focus rings |
| Destructive | Not applicable this phase | No delete/edit/destructive actions in Phase 3 (browse-only, per PERS-01ŌĆō04) |

Accent reserved for: the selected sidebar nav item (Companies OR Key Personas ŌĆö never both simultaneously, per `03-RESEARCH.md` Pitfall 3), the selected Persona list row's left-border/highlight indicator, active filter chip background, inline text links (the "Current Company" block's link to `/companies/[id]`, styled as an accent-colored link per this reservation list), and focus rings on interactive elements. **Never** apply accent to static badges, section headers, or body text.

**Contact-info links (new this phase, direct extension of the accent-link reservation above):** LinkedIn URL renders as an `<a>` using the same indigo-600 link treatment; email renders as a `mailto:` link, same treatment. Both are plain text-weight (14px/400/1.5, Body role) with indigo-600 color and underline-on-hover ŌĆö no new link style introduced.

**Signal badges:** Persona detail does not render `SignalBadge` directly (signals belong to Company per COMP-03; Persona's "has signals via current company" is a filter-only concept per D-07, not a rendered badge on the Persona list/detail). No new badge color decision needed this phase.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable this phase ŌĆö same as Company Explorer. No page-level CTA button; a Persona list-row click is the primary interaction (opens master-detail pane). Do not add a "View" button on rows. |
| Empty state heading (filtered ŌĆö no results match) | "No personas match your filters" |
| Empty state body (filtered) | "Try removing a filter or clearing your search." |
| Empty state heading (true empty ŌĆö no data loaded) | "No personas yet" |
| Empty state body (true empty) | "Persona data will appear here once the seed dataset is loaded." |
| Error state | "Couldn't load personas" ŌĆö "Something went wrong fetching this data. Try refreshing the page." |
| Destructive confirmation | Not applicable ŌĆö no destructive actions exist in this phase |
| Career History empty state (persona has no past roles) | "No career history recorded." (mirrors `company-detail.tsx`'s "No tech stack recorded." / "No buying signals recorded." pattern exactly ŌĆö same 14px/400/slate-500 body-copy treatment, inline within the section, not a full empty-state card) |
| Current Company empty state (persona has no `isCurrent` role ŌĆö edge case) | "No current company on record." (same inline treatment as above) |
| Contact Info empty state (both email and LinkedIn null) | "No contact info on record." (same inline treatment as above) |

Note: exact copy pattern for the three new inline section-level empty states above is derived by direct substitution from `company-detail.tsx`'s existing "No {X} recorded." convention (Tech Stack / Buying Signals sections) ŌĆö not a new copy pattern, same voice and structure. EXPL-06 (formal empty/loading/error hardening) remains Phase 4 scope; this baseline copy exists so Phase 3 never visibly breaks in these states, matching Phase 2's precedent exactly.
